# 证道视频中文字幕 Pipeline 可行性分析 / Sermon Video Chinese Subtitle Pipeline Feasibility Report

Report date: 2026-06-22  
Timezone used for SLA analysis: America/Los_Angeles, PT  
Target SLA: subtitles generated every Sunday by 11:30 PT, hard deadline 11:50 PT

## 中文报告

### 1. 背景

目标是建设一个数据 pipeline，从英文证道视频中自动生成中文字幕，服务看不懂英文或更习惯中文阅读的观众。

本阶段只分析源视频是否能及时进入 pipeline，不设计字幕显示方式。核心问题是：目标频道的视频是否会在每周日 11:30 PT 前已经上传并可被 pipeline 处理，最晚是否能支持 11:50 PT 前生成中文字幕。

目标视频：

- URL: https://www.youtube.com/watch?v=V6OKiwbjDZE
- Title: The Cure for Our Rebellion - Eric Geiger | Mariners Church
- Channel: Mariners Church
- Duration: 30:58

### 2. 数据来源与方法

本次分析使用公开 YouTube 页面元数据，通过 `yt-dlp` 抽取视频与频道近期公开视频信息。关键字段包括：

- `video_id`
- `title`
- `timestamp`
- `upload_date`
- `duration`
- `channel`

分析时统一将 YouTube 返回的 epoch timestamp 转换为 `America/Los_Angeles` 时间，因为业务 SLA 是周日 11:30-11:50 PT。

需要特别说明：公开侧能稳定检测的是视频何时对外可见，不能证明频道后台真实上传完成时间。如果频道提前上传并设置定时公开，公开 pipeline 看不到这个提前上传状态，除非获得频道方授权、YouTube Studio/API 权限、原始文件源，或直播/内部分发源。

### 3. 目标视频结果

目标视频 `V6OKiwbjDZE` 的公开可见时间为：

```text
2026-06-21 12:28:32 PDT
```

相对 SLA：

- 比 11:30 PT 晚 58 分 32 秒
- 比 11:50 PT 晚 38 分 32 秒

因此，仅从公开视频开始处理，该视频不可能满足 11:50 PT 前完成中文字幕的目标。

### 4. 历史发布时间分析

从 `Mariners Church` 频道近期公开视频中，筛选出 26 条周日主证道视频。筛选逻辑排除了 `If I Had More Time`、`Intentional Parenting`、`Gospel Every Day`、`On The Table`、trailer、story、short devotional 等非主证道视频，并保留周日发布、时长通常在 28-40 分钟左右的主 sermon。

统计结果：

|---|---:|
| 样本数量 | 26 |
| 11:30 PT 前公开 | 0 / 26 |
| 11:50 PT 前公开 | 0 / 26 |
| 12:00 PT 前公开 | 0 / 26 |
| 12:10 PT 前公开 | 0 / 26 |
| 12:25 PT 前公开 | 1 / 26 |
| 13:00 PT 前公开 | 23 / 26 |
| 最早公开时间 | 12:24:21 PT |
| 中位公开时间 | 12:31:48 PT |

如果排除明显异常晚发的 3 条，常规周日主证道公开视频时间大多集中在：

```text
12:28-12:43 PT
```

这说明频道公开视频节奏相当稳定，但稳定在 11:50 之后。

### 6. 可行性判断

当前公开 YouTube VOD 方案无法满足目标 SLA。

原因不是 ASR 或翻译速度，而是源视频在公开视频层面的可用时间太晚。字幕 pipeline 的输入在 12:28 PT 左右才出现，已经晚于 11:50 PT 硬截止。

可行性分级：

| 输入源 | 11:50 PT 前完成可能性 | 说明 |
|---|---|---|
| 公开视频 VOD | 不可行 | 视频通常 12:28 PT 之后才公开 |
| YouTube 定时公开前的频道方授权源 | 可行但需要权限 | 如果能提前拿到源文件或未公开视频 |
| 直播/HLS 实时源 | 可能可行 | 需要实时转写，字幕质量与延迟要评估 |
| 频道方直接提供音频/讲稿 | 最可行 | 可以提前或同步处理 |

### 7. 建议的检测 Pipeline

第一阶段只做检测和证据积累，不直接承诺字幕生成。

#### 7.1 Video Availability Detector

职责：

- 每周日 10:45 PT 开始轮询目标视频 URL 和频道公开视频列表
- 轮询间隔建议 60-120 秒
- 一旦发现目标视频公开，记录精确时间

记录字段：

- `detected_at`
- `video_id`
- `title`
- `channel_id`
- `channel_name`
- `duration`
- `public_available_at`
- `availability`
- `captions_available`
- `source_url`

#### 7.2 Historical Publish Analyzer

职责：

- 每周更新同频道历史发布样本
- 过滤主证道视频
- 计算是否能满足 SLA

关键指标：

- `before_11_30_rate`
- `before_11_50_rate`
- `median_public_time`
- `p90_public_time`
- `late_outlier_count`
- `current_week_status`

#### 7.3 SLA Gate

建议判定：

| 条件 | 状态 |
|---|---|
| 视频 11:30 前可用 | 可以启动正常字幕 pipeline |
| 视频 11:30-11:50 可用 | 高风险，只能启动极速 pipeline |
| 视频 11:50 后可用 | SLA 已失败，转为事后字幕 |
| 视频 12:15 后仍不可用 | 触发告警，记录源视频延迟 |

当前频道默认状态应标记为：

```text
public-youtube-source-cannot-meet-sla
```

### 8. 后续开发建议

建议下一步不要先做字幕显示，而是先做可重复运行的检测系统：

1. 写一个 `detect_video_availability` 脚本，能轮询频道和目标视频。
2. 写一个 `analyze_channel_publish_history` 脚本，生成历史发布时间报告。
3. 建立每周日自动任务，保存 JSON/CSV 证据。
4. 如果仍要求 11:50 前完成，开始评估替代输入源：频道方提前文件、未公开视频权限、直播实时源、讲稿。

### 9. 结论

公开 YouTube VOD 不是这个 SLA 的可行输入源。对于 Mariners Church 这个频道，主证道视频历史上没有在 11:50 PT 前公开的样本；常规公开时间约为 12:30 PT。因此，若目标是每周日 11:50 PT 前生成中文字幕，必须获得更早的视频或音频来源。

## English Report

### 1. Background

The goal is to build a data pipeline that automatically generates Chinese subtitles from English sermon videos for viewers who cannot understand English or prefer reading Chinese.

This phase only analyzes whether the source video becomes available early enough for processing. Subtitle display and rendering are intentionally out of scope for now. The core question is whether the channel's Sunday sermon video is uploaded and available before 11:30 PT, and whether Chinese subtitles can be generated by the hard deadline of 11:50 PT.

Target video:

- URL: https://www.youtube.com/watch?v=V6OKiwbjDZE
- Title: The Cure for Our Rebellion - Eric Geiger | Mariners Church
- Channel: Mariners Church
- Duration: 30:58

### 2. Data Source and Method

The analysis used public YouTube metadata extracted with `yt-dlp` from the target video and recent public videos on the same channel. Key fields included:

- `video_id`
- `title`
- `timestamp`
- `upload_date`
- `duration`
- `channel`

All epoch timestamps were converted to `America/Los_Angeles` because the operating SLA is defined as Sunday 11:30-11:50 PT.

Important limitation: from the public side, we can reliably observe when a video becomes public. We cannot prove the true backend upload completion time. If the channel uploads early and schedules public release, a public-only pipeline cannot see that private/scheduled asset unless we have channel-side authorization, YouTube Studio/API access, the original media source, or a live/internal feed.

### 3. Target Video Result

The target video `V6OKiwbjDZE` became publicly available at:

```text
2026-06-21 12:28:32 PDT
```

Relative to the SLA:

- 58 minutes and 32 seconds after 11:30 PT
- 38 minutes and 32 seconds after 11:50 PT

Therefore, starting from the public YouTube VOD, this video could not have met the 11:50 PT subtitle completion deadline.

### 4. Historical Publish-Time Analysis

From recent public videos on the `Mariners Church` channel, 26 Sunday main sermon candidates were selected. The filter excluded `If I Had More Time`, `Intentional Parenting`, `Gospel Every Day`, `On The Table`, trailers, stories, short devotionals, and other non-main-sermon formats. The remaining videos were Sunday-published main sermons, typically around 28-40 minutes long.

Statistics:

| Metric | Result |
|---|---:|
| Sample count | 26 |
| Public before 11:30 PT | 0 / 26 |
| Public before 11:50 PT | 0 / 26 |
| Public before 12:00 PT | 0 / 26 |
| Public before 12:10 PT | 0 / 26 |
| Public before 12:25 PT | 1 / 26 |
| Public before 13:00 PT | 23 / 26 |
| Earliest public time | 12:24:21 PT |
| Median public time | 12:31:48 PT |
| 90th percentile public time | 12:43:34 PT |
| Latest public time | 17:05:26 PT |

Excluding three clear late outliers, the normal public release window is approximately:

```text
12:28-12:43 PT
```

The channel is consistent, but consistently later than the 11:50 PT deadline.

### 6. Feasibility Assessment

The public YouTube VOD approach cannot meet the target SLA.

The bottleneck is not ASR or translation speed. The source video becomes publicly available too late. The pipeline input usually appears around 12:28 PT or later, already after the hard deadline.

Feasibility by input source:

| Input source | Can meet 11:50 PT? | Notes |
|---|---|---|
| Public YouTube VOD | No | Main sermons usually become public after 12:28 PT |
| Channel-authorized pre-release source | Possible, requires access | Requires early source file or private/scheduled video access |
| Live/HLS real-time source | Possibly | Requires real-time ASR; latency and quality need testing |
| Audio or transcript from the channel | Most feasible | Can be processed before or during publication |

### 7. Recommended Detection Pipeline

The first implementation phase should focus on detection and evidence collection, not subtitle rendering.

#### 7.1 Video Availability Detector

Responsibilities:

- Start polling the target video URL and channel video list every Sunday at 10:45 PT.
- Use a polling interval of 60-120 seconds.
- Record the exact time once the target video becomes public.

Fields to record:

- `detected_at`
- `video_id`
- `title`
- `channel_id`
- `channel_name`
- `duration`
- `public_available_at`
- `availability`
- `captions_available`
- `source_url`

#### 7.2 Historical Publish Analyzer

Responsibilities:

- Refresh historical publish samples for the same channel every week.
- Filter for main sermon videos.
- Recompute SLA feasibility.

Key metrics:

- `before_11_30_rate`
- `before_11_50_rate`
- `median_public_time`
- `p90_public_time`
- `late_outlier_count`
- `current_week_status`

#### 7.3 SLA Gate

Recommended statuses:

| Condition | Status |
|---|---|
| Video available before 11:30 | Start normal subtitle pipeline |
| Video available between 11:30 and 11:50 | High risk; start fast-path pipeline |
| Video available after 11:50 | SLA missed; generate post-event subtitles |
| Video unavailable after 12:15 | Alert and record source delay |

The current default status for this channel should be:

```text
public-youtube-source-cannot-meet-sla
```

### 8. Recommended Next Development Steps

The next step should be a repeatable detection system, not subtitle display:

1. Implement `detect_video_availability` to poll the channel and target video.
2. Implement `analyze_channel_publish_history` to generate historical timing reports.
3. Set up a Sunday scheduled job that stores JSON/CSV evidence.
4. If the 11:50 deadline remains required, evaluate earlier input sources: channel-provided files, private/scheduled video access, live streams, or prepared transcripts.

### 9. Conclusion

Public YouTube VOD is not a viable input source for this SLA. For the Mariners Church channel, no sampled Sunday main sermon became public before 11:50 PT. The normal public release time is around 12:30 PT. To generate Chinese subtitles by 11:50 PT every Sunday, the project needs access to an earlier video, audio, or transcript source.
