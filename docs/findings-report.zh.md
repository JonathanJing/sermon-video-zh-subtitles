# 证道视频中文字幕 Pipeline 发现报告

English version: [findings-report.md](./findings-report.md)

报告日期：2026-06-22  
分析对象：Mariners Church YouTube 频道  
目标视频：The Cure for Our Rebellion - Eric Geiger | Mariners Church  
目标链接：https://www.youtube.com/watch?v=V6OKiwbjDZE  
业务目标：每周日 11:30 PT 开始前完成源视频检测，最晚 11:50 PT 前生成中文字幕

## 一、执行摘要

我们当前的核心发现是：如果 pipeline 只能等 YouTube 公开视频出现后再开始处理，那么无法满足每周日 11:50 PT 前生成中文字幕的目标。

目标视频 `V6OKiwbjDZE` 在 YouTube 公开可见的时间是 `2026-06-21 12:28:32 PDT`，已经比 11:50 PT 晚了 38 分 32 秒。也就是说，即便转写、翻译、字幕生成全部瞬间完成，也已经错过 deadline。

进一步检查同频道近期 26 条周日主证道视频后，发现没有任何一条在 11:50 PT 前公开。频道主证道视频通常在 12:28-12:43 PT 左右公开，中位公开时间约为 12:31 PT。

因此，本阶段结论不是“字幕生成技术不可行”，而是“公开 YouTube VOD 作为输入源不可行”。如果仍要坚持 11:50 PT 前完成中文字幕，必须获得更早的视频、音频、直播流或讲稿来源。

## 二、分析问题

本次分析要回答两个问题：

1. 目标视频是否在周日 11:30 PT 前已经可用？
2. 同频道历史视频是否显示出稳定的提前公开视频规律，足以支持 11:50 PT 前完成中文字幕？

这两个问题都直接关系到 pipeline 的第一步：源视频什么时候进入系统。如果源视频本身太晚出现，后续再快的 ASR、翻译和字幕封装都无法弥补时间差。

## 五、关键限制

公开 YouTube 数据能告诉我们视频什么时候对外可见，但不能证明频道后台真实上传完成时间。

存在一种可能：频道方提前上传视频，并设置为定时公开。如果是这种情况，公开侧 pipeline 看不到提前上传完成的状态。要利用提前上传的视频，需要获得频道方授权、未公开视频访问权限、YouTube Studio/API 权限、原始文件交付，或其他内部源。

所以这里要区分两个概念：

| 概念 | pipeline 是否能公开检测 |
|---|---|
| 后台上传完成时间 | 不能稳定检测 |
| 公开视频可见时间 | 可以检测 |
| 未公开视频可访问时间 | 需要授权 |
| 直播流开始时间 | 可能可以检测，但需要单独验证 |

## 六、对 Pipeline 的影响

如果继续采用公开视频 VOD 作为输入源，pipeline 实际会变成事后字幕系统，而不是 11:50 PT 前完成的准实时字幕系统。

按当前频道节奏，合理的公开 VOD pipeline 时间线大致是：

| 时间 | 事件 |
|---|---|
| 10:45 PT | 开始轮询目标视频和频道列表 |
| 11:30 PT | 目标 deadline 前置点，但公开视频通常还不可用 |
| 11:50 PT | 硬 deadline，但公开视频仍通常不可用 |
| 12:28-12:43 PT | 主证道视频通常变为公开 |
| 公开视频出现后 | 才能开始下载、转写、翻译、生成字幕 |

因此，公开视频 VOD pipeline 的真实产出时间只能落在公开视频出现之后，无法提前到 11:50 PT。

## 七、可行路径评估

| 路径 | 是否可满足 11:50 PT | 说明 |
|---|---|---|
| 等 YouTube 公开视频 VOD | 不可行 | 源视频通常 12:28 PT 之后才公开 |
| 获取频道方提前上传文件 | 可行性高 | 最适合稳定生成高质量字幕 |
| 获取未公开视频授权访问 | 可行，但依赖权限 | 需要频道方授权和稳定访问方式 |
| 使用直播/HLS 实时转写 | 可能可行 | 需要测试直播源、延迟、ASR 质量 |
| 使用讲稿或 sermon notes | 可行性高 | 如果讲稿提前准备好，字幕生成压力最小 |
| 事后生成字幕 | 可行 | 可服务回看观众，但不满足 11:50 deadline |

## 八、建议的下一步

建议下一阶段先建设检测和证据积累系统，不急着做字幕显示层。

第一步：实现公开视频检测脚本。

- 每周日 10:45 PT 开始轮询目标视频 URL 和频道公开视频列表。
- 每 60-120 秒检测一次。
- 记录首次公开视频可见时间、标题、时长、字幕状态和检测时间。

第二步：实现历史发布时间分析脚本。

- 每周拉取同频道近期视频。
- 自动筛选周日主证道候选。
- 生成 SLA 指标：11:30 前公开视频比例、11:50 前公开视频比例、中位公开时间、90 分位公开时间、异常晚发样本。

第三步：建立 SLA gate。

| 条件 | 状态 |
|---|---|
| 视频 11:30 前可用 | 可以进入正常字幕 pipeline |
| 视频 11:30-11:50 可用 | 高风险，需要极速 pipeline |
| 视频 11:50 后可用 | SLA 已失败，转为事后字幕 |
| 视频 12:15 后仍不可用 | 触发源视频延迟告警 |

第四步：并行寻找更早输入源。

- 联系频道方，看是否能获得提前上传文件、讲稿、音频或未公开视频访问。
- 调研是否有稳定直播源可用于实时转写。
- 如果频道方无法提供更早源，则把产品目标调整为“公开视频后尽快生成中文字幕”。

## 九、结论

当前证据支持一个明确判断：对于 Mariners Church 频道，公开视频 YouTube VOD 不能作为 11:50 PT 前生成中文字幕的输入源。

这不是字幕技术的失败，而是输入源时效性的限制。后续开发应先解决源视频获取时间问题，再进入 ASR、翻译、字幕封装和显示层设计。

建议项目当前状态标记为：

```text
public-youtube-source-cannot-meet-sla
```

如果能获得频道方提前视频、音频、讲稿或直播源，项目才具备满足 11:50 PT deadline 的现实基础。
